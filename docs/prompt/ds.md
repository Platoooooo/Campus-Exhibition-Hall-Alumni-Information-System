### 【1-1】数据库初始化
```

另外生成一份最小化的初始化数据 init.sql：仅包含一个超级管理员账号（角色 admin，密码用 BCrypt 占位，注释说明明文是 admin123）和 3~4 个基础资料分类（荣誉/作品/成绩/活动经历）。

> 验收点：执行后库表齐全，能查到 admin 账号和分类。

### 【1-2】后端基础设施
```
在 /server 搭建后端基础设施，严格遵循 CLAUDE.md 的分层与规范：
1. 集成 MyBatis-Plus + MySQL（application.yml 用 campus_exhibition 库，连接信息走配置）；
2. 统一返回体 R<T>{code,message,data} 与分页对象 PageResult<T>；

### 【1-3】认证与 RBAC
```

1. 登录接口 /api/auth/login（账号密码 → 返回 JWT，密码 BCrypt 校验）；
2. JWT 过滤器 + 当前登录用户上下文（可取 userId/role/collegeId）；
3. 基于注解的接口鉴权（@PreAuthorize），三角色：college/academic/admin；
4. 数据权限思路落地：提供一个工具方法，学院管理员查询时自动注入 college_id 条件，教务处/校级不限制。
用 init.sql 里的 admin 账号验证登录可拿到 token，并用 token 访问 /api/ping。
```
> 验收点：错误密码被拒；正确登录拿到 token；带 token 可访问受保护接口。

# 给你的执行提示

1. **顺序很重要**：阶段 0→1→2→3 是硬依赖，3 之后前端(5)和人脸(4)可并行。若你想早点看到效果，可在 2-2 后插一条「先做 5-1/5-2 看后台」。
2. **每条收尾都让 Claude 给验证命令**（curl / 启动命令），你照着跑一遍再继续，能极大减少返工。
3. **face-service 真实代码**：4-1 先让它搭壳留 TODO，然后你把本地调通的 Arcface 代码贴进对话，让它替换占位实现。
4. **大屏自适应**：5-5 的"1920 基准等比缩放"很关键，一体机分辨率若是 4K，记得在 prompt 里把基准告诉它。
