# 一、功能需求文档

```markdown
# 校园展览馆校友资料系统 · 功能需求文档（SRS）

## 1. 项目概述
建设校级校友资料收集—多级审核—统一入库—大屏展示一体化平台。
支持图片/视频等多媒体资料管理，支持展览馆大屏轮播/列表/查询，
并通过一体机摄像头人脸识别定位观看人，触发对应校友信息轮播。

## 2. 系统模块总览
- 认证与权限模块
- 校友资料管理模块
- 多级审核模块
- 分类与媒体资源模块
- 校友资料库运营模块（维护/分类/上架）
- 大屏展示模块
- 人脸识别交互模块
- 系统管理模块（学院/账号/日志）

## 3. 后端功能设计（Spring Boot）

### 3.1 认证与权限
- 账号登录（JWT），密码 BCrypt 加密
- RBAC：角色（学院管理员/教务处/校级管理员），数据权限按学院隔离
- 登录态校验拦截器、接口权限注解 @PreAuthorize

### 3.2 校友基础信息
- 校友 CRUD（学号、姓名、学院、专业、入学/毕业年份、头像、状态）
- 在校生/校友标识；批量导入（Excel 模板）
- 校友检索（姓名/学号/学院/年份多条件分页）

### 3.3 资料档案（荣誉/作品/成绩/经历）
- 档案条目 CRUD，绑定校友 + 分类 + 时间点（用于成长轨迹排序）
- 一个档案可挂多个媒体附件（图片/视频/文档）
- 档案状态机：draft / pending_college / pending_academic / approved / rejected / published / unpublished

### 3.4 多级审核
- 学院审核接口：通过 / 驳回（必填意见）
- 教务处审核接口：通过（入库）/ 驳回
- 审核记录留痕（操作人、时间、动作、意见）
- 我的待办（按角色聚合待审列表）

### 3.5 分类管理
- 资料分类树（荣誉类/作品类/成绩类/活动经历…，支持自定义）
- 分类启停、排序、图标

### 3.6 媒体资源
- 文件上传（图片/视频），本地存储或对象存储
- 视频转码可选；缩略图生成；大小/格式校验
- 媒体访问鉴权（大屏端公开只读）

### 3.7 资料库运营（校级）
- 主动维护：编辑任意已入库资料
- 上架/下架；展示排序、置顶、推荐
- 轮播配置：默认轮播池、轮播时长、切换动效、播放顺序

### 3.8 大屏数据服务（公开只读 API）
- 轮播数据接口、校友墙列表接口、校友详情/轨迹接口
- 查询接口（按姓名/学院/年份/分类）
- 接口做缓存（Redis 可选），限制只返回 published 数据

### 3.9 人脸识别交互
- /face/recognize：接收一体机抓拍帧，转发 Arcface 服务比对
- 命中后返回 alumniId，大屏端拉取该校友专属轮播数据
- /face/enroll：（校级后台）为校友录入/更新人脸特征
- 识别日志（命中率、时间），隐私合规：仅存特征向量不存原图（可配置）

### 3.10 系统管理
- 学院/专业字典管理
- 账号管理（增删改、重置密码、分配学院）
- 操作日志、登录日志

## 4. 前端功能设计

### 4.1 管理后台（Vue3 + Element Plus）
- 登录页
- 工作台（待办统计、资料统计图表）
- 校友管理（列表/详情/编辑/导入）
- 资料档案（列表/编辑/媒体上传）
- 审核中心（学院待办 / 教务处待办 / 审核详情）
- 分类管理
- 资料库运营（上架管理 / 轮播配置）
- 系统管理（学院、账号、日志）

### 4.2 大屏展示端（Vue3 独立工程，无 Element 重组件）
- 默认轮播大屏（全库精选轮播）
- 校友墙（瀑布/网格列表）
- 校友详情·成长轨迹（时间线 + 媒体）
- 查询页（虚拟键盘/语音可选 + 结果展示）
- 人脸识别浮层（识别中动画 → 命中后切换专属轮播）

## 5. 非功能需求
- 性能：大屏接口响应 < 300ms，人脸比对端到端 < 1.5s
- 可用性：大屏端断网/识别异常自动降级为默认轮播
- 安全：JWT 鉴权、接口权限、文件类型白名单、人脸数据加密存储
- 兼容：管理后台 Chrome/Edge；大屏 Chrome kiosk 横屏
- 隐私合规：人脸特征单独加密、可配置不留存原图、提供采集告知

## 6. 接口风格约定
- RESTful + 统一返回体 { code, message, data }
- 分页统一 { pageNum, pageSize, total, list }
- 时间统一 ISO8601；枚举用字符串常量
